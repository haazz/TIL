# TDA (Tell Don't Ask) 원칙

객체지향의 동작 중심 설계 및 캡슐화 방식을 통해 외부에서 동작에 대한 것을 제어하지 말고, 객체에 시키는 것이 더 바람직하다.

```java

param == null ? null : param.doSomething();

public Something doSomething(Param param) {
    ...
}

```

```java

param.doSomething();

public Something doSomething() {
    if (this == null) {
        return null;
    }
    ...
}

```
