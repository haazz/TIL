Java Stream은 데이터 소스에 대한 연산을 선언적으로 연결해 처리하는 파이프라인 API이다.

# 주요 특징

- 스트림 처리 단계:
  - 스트림은 생성, 중간 연산, 최종 연산 단계로 처리된다.
  - 생성은 데이터 소스를 스트림으로 변환하는 과정으로 지연 읽기가 가능한 소스(I/O 소스)에서는 메모리 사용량을 최적화할 수 있다.
  - 중간 연산은 데이터 소스를 가공하는 단계로 입력값과 연산 결과 모두 Stream이다.
  - 최종 연산은 결과를 산출하기 위한 단계로 호출 후 스트림은 재사용이 불가능하다.
- 지연 연산 (Lasy Evaluation):
  - 스트림의 중간 연산은 즉시 실행되지 않고 파이프라인만 쌓인 상태에서 최종 연산 호출 시 전체 파이프라인이 순회되며 실행된다.
  - 이를 통해 쇼트서킷 연산이 있거나 여러 조건이 있을 때 불필요한 계산을 피할 수 있다.
- 불변·무상태 지향:
  - 불변: 스트림 파이프라인은 외부 상태를 바꾸지 않고 입력을 읽어 새 값을 만들어 반환하는 방식을 지향한다.
  - 무상태: 각 요소를 처리하는 람다는 이전 요소 처리 결과나 외부 누적 상태에 의존하지 않도록 설계한다.

# 장점

- 가독성 향상:
  - 선언적 표현과 체이닝을 통해 복잡한 로직을 간결하게 표현할 수 있다.
- 병렬처리 지원(parallel(), parallelStream()):
  - 데이터의 흐름을 나누어서 멀티 스레드로 병렬로 처리하고 합치는 과정을 통해 대량의 데이터를 빠르게 처리할 수 있다.

# 단점 및 주의점

- 작은 입력에서는 오버헤드 발생:
  - 단순한 루프는 기존 for문 방식이 더 빠를 수 있다.
- 병렬 남용 위험:
  - 작은 데이터, 가벼운 연산, 짧은 파이프라인, I/O 포함 등에서는 오히려 느려질 수 있다.
- 디버깅이 까다롭다:
  - 체인 내부를 단계별로 보기 어렵다. 필요시 peek()을 디버깅 용도로 사용
