우선 무중단 배포릉 위해서는 최소 2개의 서버와 로드밸런서가 필요하다.

# 1. 롤링 배포

- 기존 운영중인 서버 중 일부 단위씩 새버전으로 점진적 교체 히는 방식
- 서버 자원이 충분한자 검증 필요, 구신 버전간의 호환성 체크 필요 (함께 공존하는 기간이 존재)
- ElasticBeanstalk에서도 기본적으로 이 방식 사용

# 2. 블루그린 배포

- 기존버전(블루)의 모든 트래픽을 일괄적으로 신버전(그린)으로 전환하는 방식

장점

- 구신 버전간 호환성 체크 불필요
- 신버전을 미리 배포하여 실제와 동일한 환경 테스트 가능

단점

- 서버자원을 두배로 유지 필요
- 신버전에 대한 선행 테스트 필요

# 3. 카나리 배포

- 위 두 배포 방식을 합쳐놓은 방식
- 지정한 서버 혹은 사용자에게만 신버전 배포 이후 전체 배포

장점

- 위험상황 빠르게 감지
- A/B 테스트 가능

단점

- 네트워크 트래픽 제어 부담
